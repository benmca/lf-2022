# Repository Guidelines

## Project Structure & Module Organization

- `posts/` holds Markdown content. Daily “minute” posts live in `posts/1min/` and are numeric (e.g., `posts/1min/123.md`) to match `postnumber`.
- `_includes/` contains Nunjucks layouts/partials; `_includes/_styles/` is modular SCSS.
- `img/`, `thumbs/`, `episode_images/`, `snd/`, `assets/`, `js/` are static assets copied to output.
- Eleventy output is `_site/` (generated).

## Build, Test, and Development Commands

- `./serve.sh` — clean build, generate thumbnails, update MP3 metadata, then run Eleventy with a dev server.
- `npx @11ty/eleventy` — build the site once (no server).
- `./make_thumbs.sh` — generate 100x100 thumbnails from `img/` into `thumbs/` (requires ImageMagick `magick`).
- `python update_mp3_metadata.py` — fills `duration` and `length` in frontmatter from MP3s (requires `mutagen`).
- `./deploy.sh` — production build + rsync deploy (contains host-specific commands).

## Coding Style & Naming Conventions

- Indentation follows existing files: 2 spaces in JS/SCSS and short, wrapped lines in templates.
- Keep Nunjucks/Markdown frontmatter keys consistent (`title`, `date`, `tags`, `postnumber`, `eleventyComputed`).
- Minute assets follow: `img/1min/{postnumber}.jpg`, `thumbs/1min/{postnumber}.jpg`, `snd/1min/{postnumber}.mp3`.

## Testing Guidelines

- No automated tests or linting are configured.
- The `test/` directory is currently a placeholder; validate changes by running `./serve.sh` and checking rendered pages, audio players, and feeds.

## Commit & Pull Request Guidelines

- Recent commits are short, lowercase, and descriptive (e.g., “content update jan 2026”).
- Keep commits focused; include context in the PR description (what changed and why).
- For visual or content changes, include before/after screenshots or sample links.
- Note any asset or feed changes explicitly (e.g., new `posts/1min/` entries or feed template edits).

## Notes for Contributors

- This is an Eleventy (11ty) v2 + Nunjucks site; SCSS is compiled by Eleventy via `.eleventy.js`.
- `posts/1min/` content drives the calendar and playlist collections; maintain tag conventions for discovery.
